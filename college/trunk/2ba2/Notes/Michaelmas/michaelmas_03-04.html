 6. Array and String classes in Eiffel

Eiffel does not regard a string as an array of character. The STRING class offers a lot more functionality that the ARRAY class.

The ARRAY class is general in the sense that it is a Generic class. The items in an array may be from any other class.

·         Copy and Aliasing

To copy contents of B into A, use A := clone(B)

or if A is already created, use A.copy(B)

The assigment A := B, makes B an alias of the array named by A,

i.e. A and B reference the same object.

 

·         Equality of Arrays and Strings

A.is_equal(B) checks if A and B are equal item by item. The notation equal(A,B) can also be used.

The simpler A=B checks if A and B refer to the same object so it is possible to have equal(A,B) true but A=B false.

 

·         Dynamic Arrays and Strings

Assume

S := "Hello"

T := " World"

then S.append(T) resizes S so that S = "Hello World"

 

·         Indexing of Arrays and Strings

z.item(4) returns the item at index 4 in the array or string z. Strings are indexed by 1..n. Arrays are indexed from lower..upper.

·         Creating an array and string

  • If a:ARRAY[REAL] then a.make(3..10) allocates space for 8 items with default values 0, indexed from 3 to 10.

If s:STRING then s.make(8) creates a string with capacity for 8.
The string is initially empty with count = 0 but it has the capacity to hold 8 items.
To fill the string, s, with blanks use s.fill_blank.

Before a string is created it is a void string.

7. Binary Search of Arrays

If an array, A, is ordered, then A can be search using the strategy of Binary Search. If the array has n items then using binary search, the index of an item, x, can be found in (log n) time or if the item is not found, indicate in (log n) time where the item, x, should be inserted. The class for BINARY_SEARCH can be given so as to include

         
class BINARY_SEARCH [G -> COMPARABLE]
feature
        found : BOOLEAN
        index : INTEGER
   
        Search(A:ARRAY[G]; L,H:INTEGER; x:G) is
               require
                       Ordered : Is_Ordered(A,L,H)
               local
                       i,j,mid: INTEGER
               do
                       from
                               i := L-1
                               j := H+1
                       until
                               j = i+1
                       loop
                               mid := (i+j)//2
                               if a.item(mid) <= x then
                                      i:=mid
                               else -- A.item(mid) > x
                                      j:= mid
                               end
                       end
                       index := i
                       found := L <= index and then equal(A.item(index), x)
               ensure
                       Found_it: (found  implies  equal(A.item(index),x))
                       Failed: (not found implies
                                      ((L-1 <= index and index <= H) and
                                      (index = L-1 or else A.item(index) < x) and
                                      (index = H or else x < A.item(index+1))))
               end -- Search
        -- .etc
endBINARY_SEARCH

If the item, x, is not found then, Search(A,L,H,x) updates index where

A.item(index) < x and x < A.item(index+1)

and if we want to insert x, we can insert it after position index.

 

Insert Sort

Insert Sorting an array is O(n2). We traverse the array and at a typical stage we have the array slice A[1..k-1] sorted and A[k..n] unsorted. We make progress and maintain the array slice A[1..k] sorted by inserting A@k into its proper position. We then get A[1..k] sorted and A[k+1..n] remains unsorted. We use Binary Search to determine the proper location for the insertion of A@k.

Insert Sort is an in-place sort and inserting an item requires that many items are shifted right.

 

 Sorting is an essential activity in Data Processing,
e.g. in order for Binary Search to work, the array must be already ordered or sorted.
We will be concerned with the in-place sorting of arrays.

8. Sorting

Performance -- O(n2) .v. O(n * (log n))

          Algorithms for sorting can be classified according to whether they are

O(n2) -- Elementary

or O(n*log n) -- Advanced

Sort algorithms can be grouped according to strategies;

Insertion:

Insert sort -- O(n2)

Merge sort -- O(n*(log n))

Selection:

Select sort --    O(n2)

Heapsort/Treesort -- O(n*(log n))

Exchange:

Bubble/Exchange -- O(n2)

Quicksort -- O(n*(log n))

Select sort may be thought of as a worst case of Quicksort.

 

Defn of Sorting a Sequence

M = Sort(L)

º

Perm(L, M) & Ordered(M)

where

Perm(L,M)

º

(All x:T |: count(x,L) = count(x,M))

 

º

Bag(L) = Bag(M)

 

Ordered(M)

º

(All k | 1£ k<n : M(k) £ M(k+1))

 

"Extremely Slow Sort"

               To sort L, generate all Perms of L and check which one is Ordered.

 9. Quicksort

 Defn Quicksort (a sequence) -- Recursive defn

Qsort [ ] = [ ]
Qsort a : X = Qsort [ b| b ¬ X ; b £ a] ++ [a] ++ Qsort [ b| b ¬ X ; b > a]

where a : X is "a prepended to X"

 

Top Level view of Quicksort Algorithm on Arrays:

Step 1.    Partition:

  • Select an item in A for the pivot P,
  • Scan from the left until A@i ³ P
  • Scan from the right until A@j£ P
  • Exchange A@i and A@j
  • Continue until scans meet/crossover

Step 2.    Sort each section:

               Having partitioned the array, we then recursively sort each section.

 10. Find the Median (C.A.R. Hoare's 'Find' Program)

The median of an array in the item in the array such that 'half' the items in the array are less than or equal to it and 'half' the items are greater than or equal to it. If the array, A, is sorted then the median is at index (A.lower + A.upper)//2, i.e. it is the middle item in a sorted array.

The median of the folowing 7 items
                               16           12           99           95           18           87            10
is 18 as 'half' the items are £ 18 and 'half' are ³ 18.
The item 18 is the middle item in size and so is the middle item when sorted.
We can find the median of a array, A, by sorting the aray and then getting the item at position (A.lower + A.upper)//2 but sorting has best performance O(n*log n) while an algorithm ("Find") developed by C.A.R. Hoare, which can be used to find the median, has performance O(n).
