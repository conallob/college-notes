<h2>Banker's Algorithm Assignment</h2>

<p><font color="red">Please note that there was a mistake in the copy
of the assignment which was distributed. The resources released by
Thread 3 were incorrect. See below (in red) for the correct resource
release for Thread 3.</font></p>

<p><font color="red">There was also a minor mistake in Message.C where
"if (errno = EINTR)" should have been "if (errno == EINTR)". This has
been corrected in the online version.</font></p>


<p>This assignment must be implemented under UNIX on the College computers (If
you do it at home make sure that it works on the machines in College before
submitting it).</p>

<p>You are asked to build a simulator of the Banker's Algorithm as follows:
</p><ul>
<li>Create a thread to simulate each of the resource requesters in the
system. Request resources by sending messages for the attention of the
"controlling" thread. Release resources in a similar fashion. Resource
request and releases should be indicated on screen. The resources
requests/releases should be as follows:
<ul>
<li>Thread 1: (MAX 2A, 3B, 1C)  +AB 3 +CAB 3 +B 9 -AABBBC 5</li>
<li>Thread 2: (MAX 1A, 2B, 3C)  +CCC 5 +AB 6 +B 8 -ABBCCC 1</li>
<li>Thread 3: (MAX 3A, 3B, 0C)  +AAA 7 +BBB 5 <font color="red">-ABBCCC</font> 8</li>
</ul>Note these lists indicate the max resources for each process, and
the order of the requests/releases. "+AB" means that one A resource and
one B resource are requested. "-AAC" indicates that two A resources and
one B resource are to be released. A number indicates a number of
seconds to wait for (pretending to be doing some processing). Note that
each thread should terminate once it has finished its sequence of
requests, releases and waits.</li>

<li> The main thread (i.e. the initial thread) should simulate the the
central controller which allocates resources as they are/become
available. It should access requests and releases from the other
threads and in the case of requests it should reply with a message to
the requesting thread when the resource is available. No response is
required for a release. Resource allocation should be shown on screen
(including a list of remaining available resources). Note that total
available resources are 3*A, 3*B, and 3*C.</li>

<li>Sample output from the program MUST be provided, as some of the
evaluation will be based on the output. This output must come from the
version of the code you submit.</li>
</ul>

<h3>Resources provided:</h3>
<ul>
<li><a href="http://134.226.32.57/courses/ba/3ba3/Assignments/Bankers/Makefile">Makefile</a></li>
<li><a href="http://134.226.32.57/courses/ba/3ba3/Assignments/Bankers/General.h">General.h</a> and <a href="http://134.226.32.57/courses/ba/3ba3/Assignments/Bankers/General.C">General.C</a></li>
<li><a href="http://134.226.32.57/courses/ba/3ba3/Assignments/Bankers/MessageQueue.h">MessageQueue.h</a> and <a href="http://134.226.32.57/courses/ba/3ba3/Assignments/Bankers/MessageQueue.C">MessageQueue.C</a></li>
<li><a href="http://134.226.32.57/courses/ba/3ba3/Assignments/Bankers/BankersController.h">BankersController.h (public methods only)</a></li>
<li><a href="http://134.226.32.57/courses/ba/3ba3/Assignments/Bankers/memwipe">memwipe</a>.
This utility (which can be invoked by typing "./memwipe") removes all
IPC mechanisms currently allocated to you. You'll need this if your
program doesn't remove the message queue at the end of a run (e.g. if
the program crashes). Note that you can see what IPC mechanisms are
currently allocated to you by invoked the <b>ipcs</b> command.
</li></ul>


<h3>To be developed:</h3>
<ul>
<li>BankersMain.C</li>
<li>BankersController.C and the rest of BankersController.h Note that
you can alter the parameters of the methods defined in
BankersController.h if you need/want to. You can also add any other
methods and/or data that you want.</li>
<li>Any other classes you feel you need.</li>
</ul>
